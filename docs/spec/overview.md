# 概要 (Overview)

OpenCV Playground（別名: Computer Vision Playground）は、**ブラウザの中だけで OpenCV の機能を実際に試せる** サーバーレスな SPA（シングルページアプリケーション）です。
ユーザーは画像をアップロード（またはサンプル画像／Webカメラを使用）し、OpenCV のデモ（例: グレースケール変換、Canny エッジ検出、輪郭検出、透視変換）を選び、スライダーでパラメータを調整しながら **Before / After** の結果をその場で確認できます。あわせて、その処理に対応する **コード例と解説** を表示し、「OpenCV で何ができるのか」を手を動かして理解できるようにします。

- 対象: **OpenCV 4.x**
- 実行基盤: 公式 **OpenCV.js（WebAssembly ビルド）** によるブラウザ内処理
- 画像処理は **すべてクライアント側で完結**。サーバーへ画像を送信しません。
- 配信形態: 静的ホスティング（GitHub Pages / Netlify / Cloudflare Pages など）上の SPA
- 技術スタック: **React + TypeScript**

> 本ドキュメントはプロジェクト全体の目的と方針を示す概要書です。
> カテゴリ・デモの具体的な定義は再掲しません。詳細は以下を参照してください。
> - `src/data/opencvCategories.ts` … 大カテゴリ 20 件の定義
> - `src/data/opencvDemos.ts` … デモ候補 91 件の定義
> - `docs/spec/opencv-feature-catalog.md` … 機能カタログ
> - `docs/spec/data-model.md` … 上記を支えるデータモデル

## アプリの目的

OpenCV Playground の目的は、**「OpenCV で何ができるか」を読むのではなく触って理解できる場を提供すること** です。

- 画像処理 / コンピュータビジョンの各機能を、**実画像 + 即時プレビュー** で体験できるようにする。
- パラメータの変化が結果にどう効くかを、スライダー操作とリアルタイムな再描画で **直感的に学べる** ようにする。
- 各デモに **最小限の OpenCV.js コード例** と **短い解説** を添え、「試す → 仕組みを知る → 自分のコードに持ち帰る」までを一気通貫でつなぐ。
- インストールや環境構築なしに、**URL を開くだけで** OpenCV の世界に入れるようにする。

加えて、本アプリは次の 2 つを **明確に区別** することを重視します。

1. **OpenCV（本体）で何ができるか** … OpenCV 4.x が提供する機能全体。
2. **OpenCV.js でブラウザ上で実際に動かせるか** … その機能が WASM ビルドで利用可能か、要調査か、ブラウザに不向きか。

この区別をデモ一覧やバッジ表示に反映し、ユーザーが「できること」と「ブラウザで試せること」を取り違えないようにします。

## 解決する課題

OpenCV はドキュメントが充実している一方で、**初学者が「結局なにができるのか」を把握しづらい** という根本的な課題があります。

- **関数リストとテキストだけでは効果が想像できない**
  公式リファレンスは関数シグネチャ・引数・数式の説明が中心で、「この処理を画像にかけると見た目がどう変わるか」が伝わりにくい。
- **環境構築のハードルが学習の入口を塞ぐ**
  Python + OpenCV、あるいは C++ のビルドなど、最初の一歩で挫折しやすい。「ちょっと試したいだけ」のコストが高い。
- **パラメータの意味が体感できない**
  `threshold1` / `threshold2`、カーネルサイズ、`apertureSize` などの数値が結果にどう影響するかは、値を動かしてみないと腑に落ちない。
- **「ブラウザで動くか」が分かりにくい**
  Web アプリに組み込みたい人にとって、OpenCV.js での可否（対応 / 要調査 / 非推奨）の情報が散在している。
- **どの機能をどんな順で学べばよいか分からない**
  機能が膨大なため、学習導線（簡単なものから発展へ）が描きにくい。

OpenCV Playground は、これらを **「触れるデモ + Before/After + パラメータ + コード/解説 + 実装可否バッジ」** という一貫した体験で解決します。

## 想定ユーザー

| ユーザー像 | 主なニーズ | 本アプリでの価値 |
| --- | --- | --- |
| CV / 画像処理を学ぶ学生・エンジニア | 各機能の効果と使いどころを理解したい | 実画像で即試せる + コード例で学習が定着 |
| プロトタイプ前に機能を試したい開発者 | 採用前に挙動とブラウザ実装可否を確かめたい | Before/After で効果検証、OpenCV.js 可否バッジで判断 |
| 教育者・講師 | 授業や勉強会で画像処理を見せたい | インストール不要、URL 共有で全員がその場で操作 |
| デザイナー / 非専門家 | 数式抜きで「何ができるか」を知りたい | スライダー中心の直感的 UI、視覚的な結果 |

- 共通する前提として、**専門知識がなくても最初の成功体験に到達できる** ことを重視します。
- 同時に、**コード例や OpenCV モジュール名** を併記することで、専門ユーザーの学習・実装にも耐える深さを持たせます。

## 主要な利用シーン

- **学習中のちょい試し**
  「Canny ってどんな結果になる?」を、手元の写真をアップして数秒で確認する。
- **パラメータ調整の体感学習**
  しきい値やカーネルサイズのスライダーを動かし、結果がどう変わるかを観察する。
- **採用判断のための機能検証**
  自分のアプリに組み込みたい処理を、実画像で効果確認し、OpenCV.js での実装可否を見極める。
- **コードの持ち帰り**
  表示された OpenCV.js のコード例をコピーし、自分のプロジェクトに転用する。
- **授業・勉強会でのライブデモ**
  講師が画面共有し、受講者も同じ URL を開いて各自の画像で同時に試す。
- **プライバシーが必要な画像の処理**
  外部に出したくない画像（個人写真・社内資料など）を、サーバーに送らずブラウザ内だけで処理する。
- **学習導線をたどる探索**
  カテゴリ（基本操作 → フィルタ → 二値化 → エッジ → 輪郭 → 幾何変換 …）を順に試し、段階的に理解を広げる。

## サーバーを持たないSPAである理由

OpenCV Playground は意図的に **バックエンドを持たない静的 SPA** として設計しています。

- **運用コストとスケールの単純さ**
  画像処理サーバーや GPU インスタンスを持たないため、サーバー保守・スケーリング・監視が不要。アクセスが増えても CDN 配信で完結する。
- **配信が容易**
  ビルド成果物（HTML / JS / WASM / アセット）を静的ホスティングに置くだけ。GitHub Pages / Netlify / Cloudflare Pages などへ低コストで配信できる。
- **障害点が少ない**
  実行はユーザーのブラウザ内。サーバー側のダウンタイムやレート制限の影響を受けない。
- **OpenCV.js（WASM）との相性**
  重い画像処理はクライアントの WebAssembly が担うため、そもそもサーバー計算が不要。SPA + WASM の構成が自然に噛み合う。
- **学習・共有のしやすさ**
  URL を共有するだけで誰でも同じ環境を再現できる。アカウント登録やログインも不要。

> 静的ホスティング前提のため、ビルドは静的書き出し（Vite / Next.js static export 等）を想定し、ルーティングや状態は URL とクライアントストレージで完結させます。

## 画像をサーバーに送らずブラウザ内で処理するメリット

すべての画像処理を OpenCV.js でブラウザ内実行することには、明確な利点があります。

- **プライバシー保護**
  画像が端末の外に出ない。個人写真・顔・書類・社内資料などを安心して扱える。「アップロードしない」こと自体が信頼につながる。
- **アップロード待ちがない / 即時フィードバック**
  サーバーへの往復が発生しないため、パラメータを動かした結果がほぼ即座に反映される。スライダー操作とリアルタイムプレビューを成立させられる。
- **通信・ストレージコストの削減**
  画像転送やサーバー側保存が不要。大きな画像でも回線やバックエンド課金を気にしなくてよい。
- **スケーラビリティとコスト効率**
  計算負荷はユーザー端末に分散される。同時利用者が増えてもサーバー費用が線形に増えない。
- **オフライン／低速回線への耐性**
  初回ロード後はアセットがキャッシュされ、ネットワークが不安定でも処理を継続しやすい（PWA 化の余地もある）。
- **データ管理責務の最小化**
  サーバーに画像を保持しないため、保存・削除・漏洩リスクといったデータ管理上の責務を抱え込まずに済む。

> トレードオフとして、初回は OpenCV.js（WASM）本体のダウンロードと初期化が必要です。これは遅延読み込み・キャッシュ・ローディング表示で体験を担保します（「基本方針」参照）。

## MVPの考え方

MVP は **「静止画 1 枚を処理する体験」を完成させること** に集中します。

### MVP の中心
- **入力**: 画像 1 枚（アップロード / サンプル画像から選択）。
- **処理**: 代表的なデモ（例: グレースケール、ぼかし、二値化、Canny エッジ、輪郭検出、透視変換 など）。
- **出力**: **Before / After の比較表示**。
- **操作**: **スライダー中心のパラメータ調整** とリアルタイム再描画。
- **学び**: 各デモに **OpenCV.js のコード例** と **短い解説** を併記。
- **正直さ**: 各デモに **OpenCV.js 実装可否（対応 / 要調査 / 非推奨）** を明示。

### MVP の方針
- 機能の幅を広げるより、**1 枚の画像で完結する体験の質** を優先する。
  - 入力 → デモ選択 → パラメータ → Before/After → コード/解説、という基本フローを磨き込む。
- まずは **`phase-1`（静止画 1 枚の基本処理）** を中核とし、優先度の高いデモから着手する。
- データ駆動で構築する。デモは `opencvDemos.ts` の定義を読み込んで表示・実行するため、**デモ追加がデータ追加で済む** 構造にする。

### MVP に含めないもの（後続フェーズ）
- Webカメラ / 動画処理
- 物体検出 / 顔検出
- 特徴点マッチング / パノラマ合成
- DNN を用いた高度な処理

> フェーズ区分は `data-model.md` の `DemoPhase`（`phase-1` 〜 `phase-7`）に準拠します。本概要では区分の再定義は行いません。

## 将来的な拡張方針

MVP で固めた基本フローを土台に、**入力の幅** と **処理の高度さ** を段階的に拡張します。

- **入力の拡張**
  - **Webカメラ入力**（ライブ映像へのフィルタ適用）
  - **動画ファイル処理**（フレーム列への一括処理）
  - **複数画像入力**（比較 / 合成 / マッチング）
  - Canvas 上の **ユーザー描画**（ROI 指定・マーカー指定など）
- **処理の高度化**
  - 輪郭・形状・図形検出の充実
  - ヒストグラム・幾何変換・実用レシピ
  - **顔検出 / 物体検出**
  - **特徴点マッチング / パノラマ合成**
  - **DNN** を用いた高度なコンピュータビジョン
- **学習・共有体験の拡張**
  - **パラメータ・プリセットの保存と共有**（URL / クエリへの状態埋め込み）
  - デモ間の **学習導線（おすすめ順・関連デモ）** の提示
  - 処理結果画像のダウンロード / コピー
- **動作環境の最適化**
  - PWA 化によるオフライン対応
  - SIMD / マルチスレッド版 OpenCV.js の採用検討

> 拡張は **「OpenCV.js でブラウザ実装できるか」を確認しながら** 進めます。ブラウザに不向き／要調査の機能は、可否ステータスとして正直に提示し、無理に常時実行はしません。

## 提供価値

OpenCV Playground が提供する中核価値は次の通りです。

- **触って分かる**: テキストや関数リストではなく、実画像 + Before/After で OpenCV を理解できる。
- **すぐ試せる**: インストール不要、URL を開くだけ。学習の入口を限界まで下げる。
- **安心して使える**: 画像をサーバーに送らないため、機微な画像も気兼ねなく扱える。
- **学びが残る**: コード例 + 解説で「試す」から「実装する」へ橋渡しする。
- **判断できる**: OpenCV.js 実装可否を明示し、Web アプリ採用の意思決定を助ける。
- **共有できる**: 静的 SPA + URL 共有で、教育・勉強会・チーム内検証に使える。

## 基本方針

開発・設計を通じて貫く指針です。

- **公式ドキュメント準拠**
  デモの分類・関数・パラメータは OpenCV 4.x 公式リファレンスに基づく。独自解釈で機能をでっち上げない。
- **OpenCV.js の限界に正直であること**
  「OpenCV でできること」と「ブラウザで実際に動くこと」を区別し、**対応 / 要調査 / 非推奨** を隠さず表示する。誇張しない。
- **教育を第一に置いた UX**
  最小の操作で最初の成功体験に到達できること、Before/After とスライダーで直感的に学べること、解説とコードで理解が定着することを優先する。専門用語には文脈を添える。
- **パフォーマンスとメモリへの配慮**
  - 重い処理はデバウンス / スロットルし、UI の応答性を保つ。
  - OpenCV.js の `cv.Mat` 等の **明示的な解放（`delete()`）** を徹底し、WASM ヒープのリークを防ぐ。
  - OpenCV.js 本体は **遅延読み込み + キャッシュ + 初期化中のローディング表示** で初回コストを吸収する。
  - 大きな画像は必要に応じて縮小し、処理時間とメモリ使用量を抑える。
- **データ駆動・型安全**
  デモ定義（`opencvDemos.ts` / `opencvCategories.ts`）を単一の真実の源とし、TypeScript の型で UI と実行を整合させる。機能追加はデータ追加を基本とする。
- **アクセシビリティと再現性**
  キーボード操作・コントラスト・状態の URL 反映などに配慮し、誰が開いても同じ結果を再現できるようにする。
