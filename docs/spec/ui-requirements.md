# UI要件 (UI Requirements)

OpenCV Playground のフロントエンド UI 要件を定義します。
本アプリはサーバーを持たない React + TypeScript の SPA で、OpenCV.js (WASM) を用いて
すべての画像処理をブラウザ内で完結させます。ユーザーはデモ（grayscale / Canny / 輪郭検出 /
透視変換 など全 91 デモ・20 カテゴリ）を選び、画像を与え（アップロード / サンプル / Webカメラ）、
パラメータを調整し、Before / After の結果・コード例・解説を確認します。

- 対象データ型は `docs/spec/data-model.md` の `OpenCvDemo` / `OpenCvCategory` に準拠します。
- 本書は UI（画面・コンポーネント・状態）に限定し、データ定義・非機能要件は各々の仕様書を参照します。
- 用語はデータモデルに合わせます（`slug`, `inputTypes`, `outputTypes`, `parameters[].control`,
  `opencvJsSupport` など）。

---

## 1. 画面構成の全体像 (Sitemap / Routing)

### 目的
ユーザーが「機能を探す → デモを試す → 学ぶ」までを最短で辿れる導線を、URL で表現・共有できる形に整える。

### 構成要素 (画面遷移)
```
トップ (/)
 ├─ カテゴリ一覧 (/categories)
 │    └─ カテゴリ詳細 = そのカテゴリのデモ一覧 (/category/:categorySlug)
 ├─ デモ一覧 / カタログ (/demos)              … 全デモ横断・フィルタ&検索
 │    └─ デモ詳細 (/demo/:slug)               … 中核画面
 ├─ About / 使い方 (/about)
 └─ 404 / Not Found
```

### URL 設計
| パス | 内容 | 主なクエリ |
| --- | --- | --- |
| `/` | トップ | — |
| `/categories` | カテゴリ一覧 | — |
| `/category/:categorySlug` | カテゴリ内デモ一覧 | `difficulty`, `priority`, `support`, `mvp`, `q` |
| `/demos` | 全デモ横断一覧 | `category`, `difficulty`, `priority`, `support`, `mvp`, `q`, `sort` |
| `/demo/:slug` | デモ詳細 | パラメータ共有用クエリ（後述） |
| `/about` | 使い方・OpenCV.js について | — |

- `:slug` / `:categorySlug` は `OpenCvDemo.slug` / `OpenCvCategory.slug`（kebab-case）をそのまま使う。
- パラメータ共有: デモ詳細では各 `DemoParameter.id` をクエリキーにして現在値を URL に反映する。
  例: `/demo/canny-edge?threshold1=80&threshold2=160&aperture=3&sample=coins`。
  入力ソースは `source=sample|upload|camera` と `sample=<SampleImageRef.id>` で表す
  （`upload` 画像は機微情報になり得るため URL には載せず、ソース種別のみ共有する）。
- ルーティングは静的ホスティング前提（Vite / Next.js static export）。SPA フォールバック（全パスを
  `index.html` に解決）を必須とし、ハッシュルーターではなく History API ベースを基本とする。
- 静的書き出しが可能な構成では `/demo/:slug` を全 slug 分プリレンダーし、初回表示と SEO/OGP を改善する。

### 振る舞い
- グローバルヘッダー（ロゴ→`/`、カテゴリ、デモ一覧、検索、言語/JS-Python 表示切替、GitHub リンク）は全画面共通。
- パンくず: `トップ > カテゴリ名 > デモ名` をデモ詳細・カテゴリ詳細に表示。

### 状態・エラー時の扱い
- 不正な `:slug` は 404 を表示し、「デモ一覧へ」「近いデモ」への導線を出す。
- 不正なクエリ値は無視して既定値にフォールバックし、URL を正規化（不要キー除去）する。

## 2. トップページ

### 目的
初訪ユーザーに「何ができるか」を 5 秒で伝え、おすすめデモへ即座に送り込む。

### 構成要素
- ヒーロー: キャッチコピー（例「ブラウザだけで OpenCV を試せる」）+ CTA（「デモを試す」「カテゴリを見る」）。
- 注目デモ: `priority === 'S'` かつ `isMvp` のデモを 6〜8 枚カード表示（grayscale / Canny / 輪郭検出 / 透視変換 など看板機能）。
- カテゴリショートカット: 代表 8 カテゴリのタイル（`isMvpCandidate` を優先）。
- 特徴説明: 「完全ブラウザ内処理 / インストール不要 / コード例つき」を 3 カラムで。
- グローバル検索バー（`/demos?q=` へ送る）。

### 振る舞い
- カードクリックで `/demo/:slug`。カテゴリタイルで `/category/:categorySlug`。
- OpenCV.js のロードはトップ滞在中に裏でプリフェッチしてもよい（詳細ページ初回の体感を改善）。

### 状態・エラー時の扱い
- データ（`opencvDemos.ts`）は静的同梱のため通常欠落しないが、空配列時はプレースホルダを表示。

## 3. カテゴリ一覧

### 目的
20 カテゴリ全体を俯瞰し、興味のある領域に入れるようにする。

### 構成要素
- カテゴリカードのグリッド。各カードに `nameJa` / `nameEn` / `description`（要約）/ デモ件数 /
  代表モジュール（`opencvModules`）/ MVP バッジ（`isMvpCandidate`）。
- 並び順は `displayOrder` 昇順、未指定は末尾。

### 振る舞い
- カードクリックで `/category/:categorySlug`。
- ホバーで `description` 全文をツールチップ表示してもよい。

### 状態・エラー時の扱い
- デモ 0 件のカテゴリは「準備中」バッジを付け、クリック先で空状態メッセージを出す。

---

## 4. デモ一覧 (カタログ)

### 目的
全 91 デモを横断的に絞り込み・検索し、目的のデモへ素早く到達する。

### 構成要素
- フィルタバー（複数条件 AND 結合）:
  - カテゴリ（`categoryId`、複数選択）
  - 難易度（`difficulty`: beginner / intermediate / advanced）
  - 優先度（`priority`: S / A / B / C）
  - OpenCV.js 対応（`opencvJsSupport`: 5 値、第 19 節の凡例と同色で表示）
  - MVP のみ（`isMvp` トグル）
- 検索ボックス（`q`）: `titleJa` / `titleEn` / `description` / `subcategory` / `opencvFunctions[].name` を対象に部分一致（大文字小文字・全半角を正規化）。
- 並び替え（`sort`）: 優先度順 / 難易度順 / カテゴリ順 / 名前順。
- デモカード: `titleJa`、`difficulty` バッジ、`priority` バッジ、`opencvJsSupport` バッジ、
  `inputTypes` / `outputTypes` のアイコン、`description`。

### 振る舞い
- フィルタ・検索・並び替えはすべて URL クエリへ同期し、リロード・共有で再現可能にする。
- フィルタ変更は即時反映（クライアント内フィルタなので体感は即時）。検索は 200ms 程度デバウンス。
- 結果件数（「91 件中 12 件」）を常時表示。

### 状態・エラー時の扱い
- 0 件時は「条件を緩める」提案と「フィルタをクリア」ボタンを表示。
- `not-supported` / `not-suitable-for-browser` のデモも一覧には出すが、カードに注意バッジを付け、
  既定フィルタで「対応見込みあり」のみ表示する初期状態にしてもよい（凡例で説明）。

## 5. デモ詳細ページ (全体レイアウト)

### 目的
1 画面で「入力 → パラメータ調整 → Before/After → コード例 → 解説」を完結させる、本アプリの中核画面。

### 構成要素
- ヘッダー部: パンくず、`titleJa` / `titleEn`、各種バッジ（難易度・優先度・OpenCV.js 対応）、共有ボタン（現在の URL コピー）。
- 入力エリア: 入力ソース切替タブ（`inputTypes` に応じて アップロード / サンプル / Webカメラ / 複数画像 / 描画 を出し分け）。
- 比較エリア: Before / After（タブで Split View に切替）。
- パラメータパネル: `parameters[]` を `control` 種別に応じて描画。リセット可。
- 下部タブ: コード例 / 解説 / ユースケース / OpenCV 関数 / 関連デモ / 注意点。

### ワイヤーフレーム（デスクトップ）
```
┌──────────────────────────────────────────────────────────────────────────┐
│ Logo  Categories  Demos   [ 🔍 search ]            [JS|Py]  [GitHub] [?]   │ ← グローバルヘッダー
├──────────────────────────────────────────────────────────────────────────┤
│ Home > エッジ検出 > Canny エッジ検出                                        │ ← パンくず
│ Canny エッジ検出  (Canny Edge Detection)        [初級] [優先度A] [✅ 対応]  │ ← タイトル+バッジ
├──────────────────────────────────┬───────────────────────────────────────┤
│ [ Upload | Sample | Camera ]     │  Parameters                  [Reset]   │
│ ┌──────────────┬──────────────┐ │  ────────────────────────────────────  │
│ │   Before     │    After     │ │  threshold1   ●────────  [ 80 ]         │ ← slider
│ │  (input)     │  (result)    │ │  threshold2   ──────●──  [160 ]         │
│ │              │              │ │  apertureSize  [ 3 ▾ ]                   │ ← select
│ │              │              │ │  L2gradient    [○ off]                  │ ← toggle
│ └──────────────┴──────────────┘ │  ────────────────────────────────────  │
│ [ Before/After | Split View ]    │  入力: image  出力: image / overlay     │
├──────────────────────────────────┴───────────────────────────────────────┤
│ [ コード例 | 解説 | ユースケース | OpenCV関数 | 関連デモ | 注意点 ]        │ ← 下部タブ
│ ┌────────────────────────────────────────────────────────────────────┐   │
│ │ // JavaScript                                              [Copy]    │   │
│ │ let dst = new cv.Mat();                                              │   │
│ │ cv.Canny(src, dst, 80, 160, 3, false);                              │   │
│ └────────────────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────────────┘
```

### 振る舞い
- 入力・パラメータが揃うと自動で処理を実行し After を更新する（デバウンス込み）。
- パラメータ変更は URL クエリへ同期（共有可能）。

### 状態・エラー時の扱い
- OpenCV.js 未ロード時は比較エリア・処理結果に「OpenCV.js を読み込み中…」を表示し、操作は受け付けるがプレビューは保留。
- 対応ステータスが `not-supported` / `not-suitable-for-browser` のデモは、入力・処理 UI を無効化し、
  代わりに理由（`browserSupportNotes` / `limitations`）とコード例・解説のみ提供する。

## 6. 画像アップロードエリア

### 目的
`inputTypes` に `image` / `multiple-images` を含むデモで、ユーザー画像を取り込む。

### 構成要素
- ドロップゾーン（点線枠 + アイコン + 「ここにドラッグ、またはクリックして選択」）。
- 隠し `<input type="file" accept="image/*">`（`multiple-images` のときは `multiple`）。
- 取り込み後のサムネイルと「差し替え」「削除」。

### 振る舞い
- ドラッグオーバーで枠をハイライト。ドロップ / 選択で `FileReader` → `<img>` → Canvas → `cv.imread`。
- すべてブラウザ内で完結し、画像はネットワーク送信しない（プライバシー注記を近くに表示）。
- 巨大画像は処理前に最大辺（例 1600px）へ自動縮小するオプションを用意。

### 状態・エラー時の扱い
- 非対応形式・画像でないファイルはエラー文言で拒否（`accept` と読み込み失敗の両方で防御）。
- サイズ警告: 一定閾値（例 8MP 超 / 5MB 超）で「大きい画像は処理が重くなります。縮小しますか?」を表示。
- 複数画像が必要なデモで枚数不足なら「あと N 枚必要です」を明示し、処理を保留。

## 7. サンプル画像選択

### 目的
画像を持たないユーザーでもすぐ試せるよう、デモに適したサンプルを選ばせる。

### 構成要素
- `SampleImageRef[]`（`demo.sampleImages`）に基づくサムネイルピッカー（`label` 付き）。
- デモ固有サンプルが無い場合は共通サンプルセット（`docs/spec/sample-images.md` 準拠）から提示。

### 振る舞い
- クリックで `path` の画像を読み込み即処理。選択中サンプルはハイライトし、`sample=<id>` を URL に反映。
- 詳細ページ初回はデモ推奨の先頭サンプルを自動選択し、「素の結果」を即表示する。

### 状態・エラー時の扱い
- 画像読み込み失敗時はそのサムネイルにエラーアイコンを出し、他サンプルへ誘導。

---

## 8. Webカメラ入力

### 目的
`inputTypes` に `camera`（または `video`）を含むデモで、ライブ映像をフレーム処理する。

### 構成要素
- 「カメラを開始 / 停止」ボタン、`<video>` プレビュー、デバイス選択（複数カメラ時）。
- HTTPS 必須・権限についての注意文。

### 振る舞い
- 開始で `navigator.mediaDevices.getUserMedia({ video: true })` を要求し、`requestAnimationFrame`
  ループで各フレームを Canvas へ描画 → `cv.imread` → 処理 → After 描画。
- 停止で全 `MediaStreamTrack.stop()` を呼びカメラを解放（インジケータ消灯まで確認）。
- 性能のため処理解像度を抑え、必要なら間引き（毎フレームでなく N フレームごと）を許可。

### 状態・エラー時の扱い
- セキュアコンテキスト（HTTPS / localhost）以外では機能を無効化し「Webカメラには HTTPS が必要です」と表示。
- 権限拒否（`NotAllowedError`）/ デバイス無し（`NotFoundError`）/ 使用中（`NotReadableError`）を区別して文言表示。
- ページ離脱・タブ非表示・コンポーネント unmount で必ずストリームを停止（リーク防止）。

## 9. Before / After 比較

### 目的
入力と処理結果を並べ、パラメータの効果を直感的に理解させる。

### 構成要素
- 左右 2 ペイン（Before = 入力、After = 結果）。各ペインにラベルとサイズ表示。
- After ペイン右上に「画像を保存」（`canvas.toDataURL` ダウンロード）。

### 振る舞い
- パラメータ変更に追従して After を再描画。両ペインの拡大率・パン位置を同期（任意）。
- マスク・オーバーレイ等、出力種別に応じた描画は第 12 節に従う。

### 状態・エラー時の扱い
- 処理中は After にスピナー＋（可能なら）直前結果を薄く残す。
- 処理失敗時は After にエラー枠とメッセージを表示し、Before は維持。

## 10. Split View 比較 (スライダ式オーバーレイ)

### 目的
同一構図で Before/After を重ね、境界スライダで差分をピクセル単位に見比べる。

### 構成要素
- 1 つの画像領域に Before を下、After を上に重ね、縦の仕切りハンドルで After のクリップ幅を制御。
- ハンドル位置インジケータ（%）、Before/After ラベル。

### ワイヤーフレーム
```
┌───────────────────────────────────────────────┐
│  Before                          After         │
│ ████████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░ │
│ ████████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░ │
│ ███████ (input) ████┃░░░░ (result) ░░░░░░░░░░░ │
│ ████████████████████┃░░░░░░░░░░░░░░░░░░░░░░░░░░ │
│ ████████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░ │
│                     ▲                          │
│                  drag handle  ──────●────  62% │
└───────────────────────────────────────────────┘
   ← After をクリップする境界。左へ動かすほど After が広がる
```

### 振る舞い
- ハンドルはマウスドラッグ / タッチ / キーボード（←→ で 1〜5% 刻み）で操作可能。
- 出力が `mask` / `overlay` の場合は After 側に重畳結果を表示し、入力と比較しやすくする。

### 状態・エラー時の扱い
- Before/After の解像度が異なる場合は同一表示サイズへスケールし、アスペクト比を保持。
- 結果未生成時は Split を無効化し Before のみ表示。

## 11. パラメータ操作パネル (ParamPanel)

### 目的
`demo.parameters[]` を `control` 種別ごとに適切な UI で提供し、リアルタイムに結果へ反映する。

### 構成要素 (control 種別ごとの UI)
| `control` | UI | 補足 |
| --- | --- | --- |
| `slider` | スライダ + 数値表示 | `min`/`max`/`step` を反映。値ラベル併記 |
| `number` | 数値入力 | `min`/`max`/`step` でバリデーション |
| `select` | ドロップダウン | `options[]`（`label`/`value`）。OpenCV フラグ名/数値を value に |
| `toggle` | スイッチ | 真偽。`default` を初期状態に |
| `color` | カラーピッカー / HSV・RGB 範囲 | しきい値範囲は下限・上限ペアで提供 |
| `point` | 画像上クリックで点指定 | 複数点（透視変換の 4 点など）はマーカー表示・ドラッグ修正 |
| `kernel-size` | 奇数限定セレクタ/ステッパ | 1,3,5,7… のみ。偶数は最近接奇数へ補正 |

- 各コントロールに `nameJa`（補助で `nameEn`）、説明（`description`）のツールチップ、対応する
  `opencvArg`（例 `ksize`, `threshold1`）を併記し学習価値を高める。
- パネル上部に「リセット」: 全パラメータを `default` へ戻し URL クエリも初期化。

### 振る舞い
- 変更は状態へ反映 → デバウンス（連続ドラッグは ~150ms、重い処理は ~300ms）後に処理を 1 回実行。
- 各値は URL クエリ（`parameter.id` をキー）へ同期し、共有・リロードで再現。
- 依存パラメータ（例: しきい値の上下関係、`kernel-size` の奇数制約）はその場で正規化・補正。

### 状態・エラー時の扱い
- 範囲外入力は `min`/`max` にクランプし、視覚的に補正を示す。
- パラメータが処理失敗を招いた場合、直前の有効値へロールバックし、原因を結果エリアに表示。

---

## 12. 処理結果表示 (ResultView)

### 目的
`demo.outputTypes[]`（複数可）に応じ、結果を最適な形で提示する。

### 出力種別 → 表示方法 の対応表
| `outputType` | 表示方法 | 補足 |
| --- | --- | --- |
| `image` | Canvas に変換後画像 | 保存ボタン、Before/After・Split に対応 |
| `mask` | 2 値マスク（白黒 or 半透明カラー） | 入力への重畳トグル、被覆率(%) を併記可 |
| `overlay` | 入力上に線・点・矩形・輪郭を重畳 | 検出色の凡例、要素数を表示 |
| `chart` | グラフ（ヒストグラム等） | Canvas / 軽量チャート描画。軸ラベル付き |
| `numeric` | 数値（面積・周囲長・座標・スコア） | ラベル付きリスト / 表。単位明記、コピー可 |
| `text` | テキスト（クラス名・判定結果） | 強調表示。複数候補はスコア順 |
| `geometry` | 幾何情報（矩形・多角形・行列） | 整形 JSON でビューア表示、コピー可、図示と連動 |
| `video` | 連続フレーム出力 | ライブ Canvas。FPS 表示、開始/停止と連動 |

- 複数出力を持つデモはタブ or 縦積みで併置（例: 輪郭検出 = `overlay` + `numeric`）。

### 振る舞い
- 出力はすべて Canvas / DOM 上に描画し、ダウンロード（画像）/ コピー（数値・geometry）に対応。
- `cv.Mat` は描画後に `delete()` してメモリを解放（実装規約、`non-functional-requirements.md` 参照）。

### 状態・エラー時の扱い
- 処理中はスピナー。失敗時はエラー枠 + 「パラメータを戻す」誘導。
- 結果が空（検出 0 件など）の場合は「該当なし」を明示し、`numeric`/`text` は 0 件表示。

## 23. 状態管理・ローディング・エラー表示の方針

### 目的
WASM ロード・在ブラウザ処理という特性に合わせ、待ち時間と失敗を明確に伝える。

### 構成要素 / 方針
- 状態管理: URL（共有対象 = デモ slug / フィルタ / パラメータ / 入力ソース）を一次ソースとし、
  揮発的 UI 状態（パネル開閉・処理中フラグ・直近結果）はローカル状態で持つ。重い共有ライブラリは必須としない。
- OpenCV.js ランタイム: `OpencvRuntimeProvider` が `loading | ready | error` を提供。
  - `loading`: 「OpenCV.js を読み込み中…（初回のみ・約数 MB）」を比較/結果エリアに表示。操作受付は可、処理は ready 後に実行。
  - `ready`: 通常動作。
  - `error`: ロード失敗時は再試行ボタン + ネットワーク/ブラウザ要件の案内。
- 処理中: 結果エリアにスピナー。長時間（例 >300ms）のみ表示してチラつきを防ぐ。可能なら直前結果を残す。
- 失敗時: 処理例外（`cv` 例外・メモリ確保失敗等）を捕捉し、ユーザー向け文言へ変換。パラメータを直前有効値へ戻す導線を出す。

### 振る舞い
- 同一入力・同一パラメータの再計算は抑制（メモ化）。連続変更はデバウンスで最後の 1 回のみ実行。
- メモリ: 各処理後に `cv.Mat.delete()` を徹底し、長時間/ライブ処理でのリークを防ぐ（実装規約）。

### 状態・エラー時の扱い
- ネットワーク断（WASM 取得失敗）はオフライン文言で区別。SW でキャッシュ済みなら継続動作。
- 想定外例外はエラーバウンダリで画面全体クラッシュを防ぎ、「再読み込み」を提示。

## 24. アクセシビリティ配慮 (概要)

### 目的
キーボード・スクリーンリーダー利用者にも基本機能を提供する（概要レベル）。

### 方針
- キーボード操作: 入力ソース切替・タブ・フィルタ・スライダ・Split ハンドルはすべてフォーカス可能で、
  Tab 移動と矢印キー操作に対応。ボタン/スイッチは Enter/Space で作動。
- ラベル: すべてのコントロールに可視ラベル or `aria-label`（`nameJa`）を付与。スライダは
  `aria-valuemin/max/now` と単位を提供。アイコンのみのボタンにもラベルを付ける。
- 状態通知: 処理中/完了/エラーを `aria-live`（polite/assertive）で読み上げ。
- コントラスト: テキスト/バッジは WCAG AA（4.5:1）以上を目安。バッジは色のみに依存せずラベル+アイコン併記（第 18/19 節）。
- 画像: 結果 Canvas には処理内容を説明する代替テキスト（例「Canny エッジ検出の結果画像」）を提供。
- フォーカス可視: フォーカスリングを明示。モーダル/下部シートはフォーカストラップと Esc クローズに対応。
